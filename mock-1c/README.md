# Mock-1C

Локальный HTTP-сервис, который имитирует внешний контур 1С для разработки и демонстрации проекта.

Это не настоящая 1С. Сервис нужен, чтобы бэкенд мог работать с внешним источником данных по HTTP, синхронизировать данные в свою БД и не зависеть от доступа к реальной университетской системе.

## Запуск

```bash
node server.js
```

Сервис стартует на:

```text
http://localhost:8091
```

## Эндпоинты

```text
GET /onec/students/2021-301-047/profile
GET /onec/students/2021-301-047/grades
GET /onec/students/2021-301-047/schedule
GET /onec/students/2021-301-047/debts
```

Дополнительные demo-студенты:

```text
2021-301-048
2022-402-015
2020-104-033
```

## Роль в проекте

Поток данных:

```text
mock-1c
  -> student-api
    -> синхронизация в БД
      -> REST-эндпоинты кабинета
        -> React фронтенд
```

Такой подход похож на реальную интеграцию: личный кабинет не обращается к внешней системе напрямую, а работает с локальной БД приложения. Внешний источник нужен только для периодического обновления данных.
